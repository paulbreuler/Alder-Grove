# .docs — Internal Documentation

Private documentation not intended for end users. This directory is
git-tracked but excluded from published artifacts.

## Structure

```
.docs/
  ├── superpowers/
  │   ├── plans/    # Implementation plans (from writing-plans skill)
  │   └── specs/    # Design specifications (from brainstorming skill)
  └── README.md
```

## Conventions

- **Specs**: `YYYY-MM-DD-<topic>-design.md` — written by brainstorming skill
- **Plans**: `YYYY-MM-DD-<topic>.md` — written by writing-plans skill
- Research and reference docs go at the `.docs/` root level
