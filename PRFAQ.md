## Press Release

**FOR IMMEDIATE RELEASE**

### Alder Grove Launches Grove: The Desktop OS Where Small Teams Grow Reliable AI-Powered Applications

**Fargo, ND — March 13, 2026** — Alder Grove today announced the launch of Grove,
a desktop-native operating system purpose-built for AI-assisted software
development. Designed for startups, solo developers, and small teams, Grove replaces
the fragmented mess of issue trackers, scattered documentation, guardrail files, and
disconnected AI terminals with a single, fast, native application where everything
stays connected and alive.

"AI agents have made coding faster than ever — but most teams are drowning in
context switching, stale specs, and technical debt," said Paul Breuler, creator of
Alder Grove. "We built Grove because the cycle should feel alive: your personas
should inform your journeys, your journeys should shape executable specifications,
those specs should guide your agents, and your actual codebase should feed back into
the system. No more drift. No more blind agents."

#### The Problem

Modern AI development is incredibly powerful yet dangerously fragmented. Teams define
personas in one place, journeys in another, specifications in a third, scatter
guardrails across repo files, and run agents in multiple terminals. Vibe-based coding
creates beautiful demos that quickly turn into unmaintainable messes. Debugging loops
waste time and money. When the code changes, the documentation dies. AI agents
operate without full context. Small teams end up spending more time realigning
everything than actually building.

#### The Solution

Grove is a lightning-fast desktop application (built on Tauri v2) backed by a cloud
API. It serves as the connected system of record for the entire software lifecycle.
Teams define **who** they are building for (Personas), **what** users need to do
(Journeys and Steps), **how** it should work (Executable Specifications with
acceptance criteria and tasks), and **what rules** AI agents must follow — all in one
living environment.

Grove connects directly to your repositories, creates intelligent Snapshots of your
actual codebase, and orchestrates AI agents through the Agent Communication Protocol
(ACP) with configurable human approval gates at every critical point.

#### Key Capabilities

- **Living Personas & Journeys** — Dynamic, connected entities that evolve and
  directly feed specifications and agents, keeping user intent alive throughout the
  lifecycle.

- **Executable Specifications** — Specs with clear acceptance criteria and tasks that
  AI agents can act upon directly, creating full traceability and preventing drift.

- **Codebase Intelligence** — Repository-connected Snapshots that reverse-engineer
  and surface what your code actually does, turning forgotten knowledge into
  structured, updatable specifications.

- **Gated AI Agent Sessions** — Orchestrates agents via ACP with mandatory human
  approval checkpoints and standardized team guardrails — eliminating blind
  prompting, debugging loops, and credit waste.

- **Native Desktop Experience** — Instant launch, zero browser latency, and seamless
  workflow alongside your editor — built for focus and speed.

Grove is available now for startups, solo developers, and small teams who want to
build software that lasts.

## Frequently Asked Questions

### Competitors

**Q: How does Grove compare to Lovable?**

Lovable is excellent at what it does: turning a single natural-language prompt into a
working full-stack prototype (React + Supabase/Stripe/etc.) in minutes. It's the
fastest way for non-technical founders and solo builders to validate ideas and ship
quick MVPs.

That speed comes with well-documented trade-offs in 2026. Users repeatedly cite
unpredictable credit burn (debugging one bug can cost more than the initial build),
endless looping problems (AI "fixes" something, re-breaks it, and keeps charging),
messy generated code that creates technical debt, and a hard ceiling once you move
past simple CRUD or landing pages. The code often feels like a black box,
traceability disappears immediately, and ownership is limited.

Grove solves exactly those frustrations. Instead of one-shot vibe coding, Grove
enforces a living, traceable cycle from Personas → Journeys → Executable Specs →
Gated Agents → real repo Snapshots. You get the speed of AI without the credit
waste, without the debugging loops, and without the debt. Your code stays clean,
owned, and connected to real user intent forever.

---

**Q: How does Grove compare to Augment Intent (from Augment Code)?**

Augment Intent is one of the strongest spec-driven, multi-agent orchestration tools
available today. It runs as a macOS-native workspace, uses a coordinator + specialist
agents, living specs, and isolated git worktrees. Developers who want structured
agentic development and solid large-codebase context love it.

Grove builds on that foundation and goes meaningfully further in three directions:

- **Upstream**: Full living Personas and Journeys (most tools stop at specs).
- **Downstream**: Deeper Codebase Intelligence via reverse-engineered Snapshots that
  turn actual code behavior into updatable structured knowledge.
- **Experience & reach**: True cross-platform native desktop (Tauri v2) with zero
  browser tabs, plus the Agent Communication Protocol (ACP) for standardized,
  team-wide guardrails and approval gates.

If Augment Intent is the conductor, Grove is the entire concert hall — with better
long-term memory and no platform lock-in.

---

**Q: What about Cursor? Isn't it the best AI coding tool right now?**

Cursor is currently the gold-standard AI-first code editor (a deeply evolved VS Code
fork). Its Composer multi-file editing, deep repository indexing, inline AI, and new
Automations make it unbeatable for day-to-day coding, refactoring, and individual
productivity.

Grove is deliberately not an editor replacement — it is designed to sit right next to
Cursor (or any editor you love).

- **Cursor** excels at the *"how do I write/edit this code right now"* layer.
- **Grove** owns the *"why are we building this, what should it actually do, and does
  it stay aligned"* layer.

Grove maintains the connected system of record (Personas, Journeys, living Specs,
Snapshots, gated agent sessions) so your agents and your editor always have perfect,
traceable context. Many power users will run both: Cursor for the editing experience
+ Grove as the persistent brain and single source of truth.

---

**Q: Why not just stitch together Lovable + Cursor + Augment Intent + Linear + Notion?**

That exact combination is what Grove was created to eliminate. Every extra tool
multiplies context switching, stale artifacts, and manual sync work. Grove collapses
the entire fragmented toolchain into one native, always-connected desktop
application. The cycle stays alive automatically — no more praying your docs match
your code or your agents know what your users actually need.
